◦ 프로젝트 개요
⁃ 한 학기 동안 학습한 Android 핵심 기술을 실제 앱으로 구현하는 개인 프로젝트 (50점 만점)
⁃ 주제: '여행 계획 및 기록 앱‘ : 사용자가 다녀온 여행지를 기록·조회·수정·삭제하고, 사진과 메모를 함께 관리
⁃ UI 디자인 및 세부 기능 구성은 자유
항목 내용
제출 방식 .apk 파일 및 개인 GitHub Repository URL 제출
제출 기한 6월 15일 오후11:59 까지 (약 4주)
최소 SDK API 26 이상
필수 구현 기술
요소
• Fragment 최소 2개 이상 사용, 백스택 관리 포함
• RecyclerView 반드시 활용 및 클릭 이벤트 처리 필수
• SQLite 필수, CRUD 전체 동작
• 옵션 메뉴 항목 2개 이상 필수 구현, 컨텍스트 메뉴 필수 구현
• 카메라 또는 갤러리 선택 Intent 필수 구현
• 지도 API 활용(지도단순구현: 가산점+1, 사진 GPS 추출 + 지도 마커 생성 가산점 +2)
• 스레드/코루틴 활용 비동기 처리 구현시 가산점 +2) (가산점의 경우 중간고사 점수, 과제 점수 및 출석 점수 감점을 채울
수 있으며, 중간/기말/출석 점수 모두 만점일 경우 적용이 되지 않을 수 있음)

◦ 필수 구현 요구사항 상세
⁃ 아래 항목은 전원 공통 필수이며, 채점 시 각 항목의 동작을 개별 확인
⁃ 모든 기능은 실제 기기/에뮬레이터에서 오류 없이 동작해야 한함
1) 여행 기록 목록 (RecyclerView): Adapter, ViewHolder 직접 구현, 여행기록들의 목록 표시
1. 여행지명, 날짜, 대표사진 또는 썸네일 포함
2. 기록은 새롭게 추가되고 삭제될 수 있고 DB랑 연동되어야함
3. 항목 클릭 시 해당 기록의 상세 화면으로 이동
2) 기록 추가/수정 : 기록 추가 및 수정화면을 별도 Activity로 분리하여 구현
3) SQLiteOpenHelper를 상속한 DBHelper 클래스를 직접 구현
1. 여행 기록의 생성·조회·수정·삭제(CRUD)를 모두 SQLite로 처리
2. 앱을 완전히 종료해도 데이터는 그대로 유지되어야 함
3. 테이블 설계 예시)

4
◦ 필수 구현 요구사항 상세
⁃ 아래 항목은 전원 공통 필수이며, 채점 시 각 항목의 동작을 개별 확인
⁃ 모든 기능은 실제 기기/에뮬레이터에서 오류 없이 동작해야 한함
1) 홈 화면 구성: 최소 2개 이상의 Fragment로 화면을 구성하여 BottomNavigationView 또는 버튼을 통한 Fragment 전환을 구현
2) 카메라 촬영 또는 갤러리 선택 인텐트로 사진 기록할 수 있는 기능 구현 (상세 화면에서 선택 사진 보여야 함)
3) 옵션 메뉴 + 컨텍스트 메뉴를 활용
1. 옵션 메뉴 항목을 2개 이상 구현 (예: 전체 삭제, 정렬 기준 변경, 앱 정보)
2. 컨텍스트 메뉴를 1개 이상 구현 (예: 목록 항목 길게 눌러 ‘삭제’ / ‘수정’)
3. 삭제 동작은 AlertDialog로 사용자 확인을 거친다
◦ 추가 구현 (가산점) 상세
⁃ AI 활용이 허용되는 만큼, 진도 미포함 항목 구현 시 가산점 부여 (없어도 만점은 가능)
1. 지도 api 활용해 지도 띄우기 +2점
2. 사진 gps 추출 + 지도 마커 생성 +4점 (api 종류 무관)
3. 스레드 및 코루틴 활용 비동기 처리(예시: 이미지 로딩 시 ProgressBar 적용 등)
